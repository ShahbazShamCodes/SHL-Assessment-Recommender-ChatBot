 SHL Assessment Recommender is an AI-powered RAG application built with FastAPI, LangChain, Mistral AI, and ChromaDB. It retrieves SHL assessment data from a vector database to provide accurate, context-aware recommendations, assessment comparisons, and conversational responses based solely on the SHL assessment catalog. SHL Assessment Recommender – Design & Implementation Approach

Project Overview The objective of this project was to build a conversational SHL Assessment Recommender that helps recruiters identify suitable SHL assessments through natural language conversations rather than relying on keyword searches or manual filtering. The system is designed using a Retrieval-Augmented Generation (RAG) architecture to ensure that all recommendations are grounded in the SHL Individual Test Solutions catalog. The application exposes a FastAPI service with a health endpoint and a chat endpoint. The chatbot accepts conversation history, retrieves relevant assessments from the SHL catalog, and generates recommendations using a Large Language Model (LLM).
Design Choices The project follows a modular architecture to separate responsibilities and improve maintainability. Technology Stack FastAPI for REST API development Python as the primary programming language LangChain for RAG orchestration ChromaDB as the vector database Mistral AI (via LangChain) for response generation JSON as the structured knowledge source Architecture SHL Catalog → JSON Loader → Text Chunking → Embeddings → ChromaDB → Retriever → Prompt Template → LLM → FastAPI Response This modular structure allows individual components such as retrieval, prompting, and recommendation logic to be improved independently without affecting the entire application.
AI Tools Used AI-assisted development tools were used during implementation to improve development efficiency rather than to generate the complete solution. These tools were primarily used for: Understanding LangChain and ChromaDB integration. Debugging FastAPI and Python errors. Reviewing project architecture. Improving prompt design. Explaining library usage and API behavior. Refactoring code into a modular structure. All project-specific logic, retrieval pipeline, integration, testing, and debugging were implemented and verified within the project environment.
